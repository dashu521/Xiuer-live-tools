`v1.3.3` 已完成版本号提升、发布文档同步、发布阻断修复回归验证和 macOS 双架构打包。与 `v1.3.2` 初次评估相比，原先的代码类 blocker 已基本清除，当前 `release:guard` 只剩 1 个阻断项：`Git 工作区不干净`。

1. 当前工作区仍包含大量未提交修改，不能安全打 tag 或直接执行正式发布。

因此，本版本状态应定义为：`版本可发布，待收敛工作区后执行发布`。

## 五、剩余风险与建议

### 阻断项

1. `Git 工作区不干净`
   - 影响：无法确认发布内容边界，不适合直接打 tag 或上传正式产物。
   - 建议：先整理工作区，只保留 `v1.3.3` 计划发布内容，完成提交后再执行正式发布流程。

### 非阻断但需处理

1. macOS 包未签名、未公证
   - 影响：部分用户首次安装时可能需要手动放行。
   - 建议：在发布说明中明确安装方式，必要时补充“右键打开”或移除隔离属性的指引。

2. `pytest` 仍有 6 条 `PytestReturnNotNoneWarning`
   - 影响：不阻断当前测试通过，但说明测试写法需要清理。
   - 建议：将返回值改为显式断言，消除 warning。

3. `release:guard` 仍提示 4 条 warning
   - 影响：当前不阻断，因为发布环境变量已正确注入；但说明仓库中仍保留若干开发态 fallback。
   - 建议：正式发布前人工复核相关 fallback 是否都只用于开发态。
