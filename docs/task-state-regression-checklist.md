# 任务状态回归测试清单

## 使用说明

本清单用于验证任务状态管理体系的稳定性。每次发布前或状态相关代码修改后，应按本清单进行回归测试。

---

## 一、自动回复手动启动/停止

### 1.1 正常启动
- [ ] 进入自动回复页面
- [ ] 点击"开启任务"按钮
- [ ] **预期结果**：
  - 按钮变为"停止任务"
  - 左侧绿点亮起
  - 日志显示 `Task comment-listener started successfully`
  - 评论开始被监听

### 1.2 正常停止
- [ ] 点击"停止任务"按钮
- [ ] **预期结果**：
  - 按钮变为"开启任务"
  - 左侧绿点熄灭
  - 日志显示 `任务已停止`
  - 评论停止被监听

### 1.3 快速重复启动
- [ ] 快速连续点击"开启任务"按钮 3 次
- [ ] **预期结果**：
  - 第一次正常启动
  - 第二次提示"任务已在运行中"（toast.info）
  - 第三次提示"任务已在运行中"
  - 无异常报错
  - 绿点保持亮起

### 1.4 快速重复停止
- [ ] 启动任务后，快速连续点击"停止任务"按钮 3 次
- [ ] **预期结果**：
  - 第一次正常停止
  - 第二次无异常（幂等）
  - 第三次无异常（幂等）
  - 绿点保持熄灭

---

## 二、开播自动启动

### 2.1 正常开播自动启动
- [ ] 开启"开播自动一键启动任务"
- [ ] 开始直播
- [ ] **预期结果**：
  - 自动回复自动启动
  - 自动发言自动启动
  - 自动弹窗自动启动
  - 三个绿点都亮起
  - 日志显示三个 `started successfully`

### 2.2 开播后自动回复失败重试
- [ ] 模拟网络异常或平台连接失败
- [ ] 开始直播
- [ ] **预期结果**：
  - 第一次启动可能失败（日志显示错误）
  - 系统自动重试
  - 最终启动成功
  - 绿点最终亮起

### 2.3 关播自动停止
- [ ] 正在运行的任务
- [ ] 结束直播
- [ ] **预期结果**：
  - 所有任务自动停止
  - 所有绿点熄灭
  - 日志显示 `activeTasks cleared`

---

## 三、失败后再次启动

### 3.1 启动失败后重试
- [ ] 模拟启动失败（如断网）
- [ ] 点击"开启任务"
- [ ] 等待失败提示
- [ ] 恢复网络
- [ ] 再次点击"开启任务"
- [ ] **预期结果**：
  - 第一次失败，绿点不亮
  - 不提示"任务已在运行中"
  - 第二次正常启动
  - 绿点亮起

### 3.2 多次失败后的状态恢复
- [ ] 连续模拟 3 次启动失败
- [ ] 第 4 次正常启动
- [ ] **预期结果**：
  - 前 3 次都失败，无"假运行"状态
  - 第 4 次正常启动
  - 绿点正常亮起

---

## 四、绿点一致性

### 4.1 waiting 状态不亮绿点
- [ ] 启动自动回复
- [ ] 在 `waiting` 状态（启动中但还未 listening）
- [ ] **预期结果**：
  - 绿点不亮
  - 进入 `listening` 后绿点亮起

### 4.2 失败后绿点不亮
- [ ] 模拟启动失败
- [ ] **预期结果**：
  - 绿点不亮
  - 不显示"假运行"状态

### 4.3 停止后绿点熄灭
- [ ] 运行中的任务
- [ ] 点击停止
- [ ] **预期结果**：
  - 绿点立即熄灭
  - 无延迟

## 六、stopAll 幂等

### 6.1 第一次 stopAll
- [ ] 启动多个任务（自动回复、自动发言、自动弹窗）
- [ ] 点击"停止所有任务"
- [ ] **预期结果**：

- [ ] **预期结果**：
  - 第一次正常停止
  - 后续 4 次幂等，无异常
  - 无状态漂移

## 十、日志检查项

测试过程中应检查日志是否包含以下关键信息：

### 正常场景
- [ ] `Task comment-listener started successfully`
- [ ] `Task auto-comment started successfully`
- [ ] `Task auto-popup started successfully`
- [ ] `任务已停止`
- [ ] `activeTasks cleared`

### 异常场景
- [ ] `任务因错误中断`（异常被抛出）
- [ ] `already stopped or not running, skipping IPC call`（幂等）
- [ ] `Task ${taskId} is already running`（ALREADY_RUNNING）

### 不应出现
- [ ] `Found stale task instance`（不应有残留）
- [ ] `Status mismatch`（状态应一致）
- [ ] 重复 `stopped` 事件
