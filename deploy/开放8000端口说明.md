# 开放 8000 端口（仅限旧应急链路排障）

> 说明：正式线上入口已经统一为 `https://auth.xiuer.work`。本说明仅用于排查旧应急链路或服务器直连问题，不再代表正式生产方案。

当界面提示「无法连接认证服务器」且控制台为 `fetch failed`、`requestUrl: "http://<legacy-ip>:8000/login"` 时，多半是**本机访问不到旧应急链路的 8000 端口**。

## 1. 在阿里云放行 8000 端口

1. 登录 [ECS 控制台](https://ecs.console.aliyun.com/)
2. 找到旧应急实例 **<legacy-ip>**，点击实例 ID 进入详情
3. 点击「安全组」→ 进入该实例使用的安全组
4. 在「入方向」里点「手动添加」：
   - **端口范围**：8000/8000
   - **授权对象**：0.0.0.0/0（或只填你本机公网 IP，更安全）
   - **描述**：auth-api
5. 保存后等几秒生效

## 2. 在本机验证

在终端执行：

```bash
curl -s http://<legacy-ip>:8000/health
```

若返回 `{"ok":true}`，说明网络已通，再在应用里重试登录或「手机验证码登录」。

## 3. 若仍不通

- 确认本机网络能访问公网（未禁 8000 出站）
- 若在公司/学校网络，可能有出口防火墙，可换手机热点试
- 确认 ECS 实例为「运行中」且 Docker 内 api 容器为 Up：  
  `docker compose ps` 中 api 应为 Up
