# 代码提交规范

## 核心原则：修改即提交

### 必须提交的场景
1. **完成一个独立功能后** - 立即 `git add + git commit`
2. **调试通过后** - 立即提交，不要等"全部完成"
3. **切换任务前** - 必须先提交当前修改
4. **长时间工作后**（每30分钟）- 阶段性提交

### 禁止的操作
- ❌ 大量修改后不提交
- ❌ 使用 `git stash` 代替提交
- ❌ 直接切换分支而不保存
- ❌ 依赖 Vite HMR 作为"保存"

### 推荐命令
```bash
# 快速提交（使用 Trae 自动生成的提交信息）
git add .
git commit -m "feat: 完成功能X"

# 或者使用 VSCode 的 Git 面板一键提交
```

### 提交信息规范
```
feat: 新功能
fix: 修复bug
docs: 文档更新
style: 代码格式（不影响功能）
refactor: 重构（既不是feat也不是fix）
perf: 性能优化
test: 测试相关
chore: 构建过程或辅助工具的变动
```
